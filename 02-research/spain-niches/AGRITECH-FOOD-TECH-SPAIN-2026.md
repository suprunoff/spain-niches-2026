## НИША 1: PAC Subsidy Automation

### Описание проблемы

Испанские фермеры подают заявки на EU CAP через сложный бюрократический процесс. Большинство используют агентства (gestoras), которые берут комиссию 2-5% от субсидии.

- **622 400 заявителей в 2023**
- Процесс подачи: 3-6 месяцев (вручную)
- Ошибки в заявках → отказы или задержки выплат
- Роль gestoras: обработка документов, подача, отслеживание статуса

### Возможность SaaS

**Гипотеза:** Автоматизировать заполнение CAP-заявок через web-форму + интеграцию с системой FEGA (Фонд гарантий для сельского хозяйства).

**Функции MVP:**
- Guided wizard для заполнения заявки (вопросы на испанском)
- Автопроверка против требований CAP 2023-27
- Интеграция с данными о размере участка (кадастр?)
- Отслеживание статуса в real-time
- Поддержка на WhatsApp (обязательно!)

**Go/No-Go метрики:**
- ✅ TAM: 622K фермеров × €1500 комиссия (в среднем) = €932M потенциала
- ✅ Конкурентов: 0 специализированных SaaS (gestoras = ручной труд)
- ✅ Цена: €300-600/год за фермера (экономит 2-4% от субсидии)
- ✅ Регуляторика: EU требует электронной подачи, нет запретов на SaaS
- ❓ Интеграция с FEGA: нужно уточнить API / доступ

### Скоринг: 8.5/10

| Критерий | Балл | Обоснование |
|----------|------|-------------|
| **Размер рынка** | 9/10 | 622K фермеров, €4.8B бюджета |
| **Urgency (боль)** | 9/10 | Сложный процесс, потери из-за ошибок, комиссии gestoras |
| **Конкуренция** | 8/10 | Gestoras — ручной труд, 0 SaaS, fragmented |
| **Цена** | 7/10 | Фермеры платят комиссии, но бюджеты ограничены |
| **Цифровизация** | 7/10 | 30% не имеют интернета, но молодые фермеры активнее |
| **Регуляторика** | 8/10 | Поддержка EU, обязательная электронизация |
| **Капитал** | 8/10 | Госбюджет, стабильный источник дохода |

**Go:** ✅ Нужна валидация на 20-30 фермерах

### Следующие шаги (CustDev)

1. Найти молодых фермеров через Linkedin/WhatsApp
2. Вопросы: "Как часто вы подаёте CAP-заявки? Сколько платите gestoras? Какие ошибки были?"
3. Показать макет wizard → feedback
4. Оценить готовность платить €400/год

## НИША 2: Wine Label Compliance SaaS (QR-коды)

### Описание проблемы

**Регулирование (Regulation EU 2021/2117):**
- С 08.12.2023 все вина должны иметь QR-код на этикетке
- QR → веб-страница с ингредиентами, аллергенами, энергией (kJ/100ml)
- Требования: на этикетке — алкоголь%, аллергены, объём, энергия
- На QR-странице — факты (БЕЗ маркетинга, БЕЗ трекинга)

**Проблема для bodeg:**
- Нужно перепечатать всё (новая этикетка + QR)
- Нужно создать веб-страницы (ISO-совместимые, без отслеживания)
- Боязнь штрафов: в Германии до €50K за нарушение
- Большинство бодег (особенно малые) ещё не соответствуют

### Возможность SaaS

**MVP: "Wine Label Maker"**

1. Загрузить инструкцию по вину (PDF, фото этикетки, ингредиенты)
2. Система генерирует:
   - Дизайн этикетки (QR + обязательные данные)
   - Печатный файл (PDF для типографии)
   - Веб-страницу (без аналитики, одноразовая ссылка)
3. Интеграция с типографией (печать + отправка)
4. Отслеживание: сколько вин напечатано, когда истекает

**Ценообразование:**
- B2C (малые бодег): €200-500/год (10-50 SKU)
- B2B (сеть ресторанов): €2000-5000/год (200+ SKU)

### Скоринг: 8.1/10

| Критерий | Балл | Обоснование |
|----------|------|-------------|
| **Размер рынка** | 8/10 | 3000+ бодег, €2.9B экспорт, €180M внутренний рынок |
| **Urgency** | 9/10 | Регулирование уже в силе (дек 2023), штрафы угрожают |
| **Конкуренция** | 8/10 | 3-4 глобальных игрока (Scantrust, QRFox), нет локального |
| **Цена** | 8/10 | Бодег платят за переделку этикеток в любом случае |
| **Цифровизация** | 7/10 | Средний уровень (50% уже используют CRM/ERP) |
| **Регуляторика** | 9/10 | Обязательно по закону, дедлайн дек 2023 (прошёл), неоднократные проверки |
| **Техсложность** | 7/10 | Средняя (дизайн + PDF-генерация + веб-хостинг) |

**Go:** ✅ Высокий потенциал, быстрая валидация

### Валидация: 3-5 бодег

- "Как вы реализовали QR-код?" → найти adopters
- "Готовы ли вы платить за автоматизацию?"
- Показать макет → feedback

## НИША 3: B2B Food Supplier Marketplace (Рестораны + Производители)

### Описание проблемы

**Глобальный рынок B2B food:** USD 47.63B (2025) → USD 203.96B (2034), CAGR 17.5%

**В Испании:**
- 40 000+ ресторанов, бара, столовых
- 50 000+ производителей (jamón, queso, aceite, etc.)
- Процесс закупок: звонки, факсы, WhatsApp, медленно
- **Проблема:** Мелкие производители не имеют доступа к рестораны (нет каналов дистрибуции)

**Уникальная возможность:**
- Испания производит лучший jamón ibérico, queso manchego, aceite virgen extra в мире
- Экспортирует €3.3B в год (вино, масло, мясо)
- Но локально: рестораны = большие оптовики, маленькие производители = нет доступа
- **Blue Ocean:** Прямая связь производитель → ресторан (без посредников)

### MVP: "Proveedores.es" (рабочее название)

**Функции:**
1. Производители:
   - Загрузить каталог (JPG/PDF → автопарсинг)
   - Ценовые условия, MOQ, доставка
   - Оценки от ресторанов (как на Trustpilot)

2. Рестораны:
   - Поиск по типу продукта (jamón, queso, aceite)
   - Сравнение цен, рейтинги
   - One-click заказ → отправка в WhatsApp поставщику
   - История заказов, рециклинг заказов

3. Платежи:
   - SEPA для B2B (безопасно)
   - Bizum для малых платежей (консумеры)

4. Инструменты:
   - Аналитика для производителей (кто смотрел, кто добавил в корзину)
   - Система рейтингов (качество, доставка, коммуникация)

### Скоринг: 7.8/10

| Критерий | Балл | Обоснование |
|----------|------|-------------|
| **Размер рынка** | 9/10 | 40K ресторанов × €200K/год закупок = €8B, B2B marketplace CAGR 17.5% |
| **Urgency** | 7/10 | Боль умеренная (работают, но медленно), не критична |
| **Конкуренция** | 6/10 | Существуют: uFoodin (EU), Mercur (B2B), но не специализированы на Испании |
| **Цена** | 8/10 | Комиссия 2-5% от заказа, или подписка €100-300/мес |
| **Цифровизация** | 6/10 | Рестораны используют POS, но закупки всё ещё аналоговые |
| **Регуляторика** | 8/10 | Без限制 (GDPR, LSSI, стандартные) |
| **Капитал для MVP** | 7/10 | Нужна масса на обе стороны (chicken-and-egg) |

**No-Go:** ⚠️ Требует двусторонней валидации + сложный запуск

### Валидация (если интересно)

1. Интервью 10-15 ресторанов Мадрида (El Corte Inglés, мал рестораны)
   - "Как вы закупаете jamón ibérico?" → боль?

2. Интервью 5-10 производителей

## НИША 5: Food Traceability SaaS (EUDR/Farm-to-Fork)

### Контекст

**Регулирование (2025-2026):**
- **EUDR (Deforestation Regulation):** Productos с риском дефорестации (какао, кофе, масло пальмы) должны быть 100% traceable
- **Farm-to-Fork:** Электронное документирование опрыскиваний, геолокация участков
- **EU Spray Records:** Со 01.01.2026 — все отчёты в электронном виде

**Проблема:**
- Фермеры не имеют цифровых систем для отслеживания
- Экспортёры должны докажите отсутствие вырубки (гео-данные, legal docs)
- Хранение в blockchain для прозрачности?

### Возможность

**MVP: "TraceX Spain"** (clone-and-adapt)
- Аганты входят в систему (photo + GPS точка)
- История всех операций (удобрения, инсектициды, вода)
- Экспорт для EUDR compliance (доказательство no-deforestation)
- Интеграция с SENASA (животные) опциональна

### Скоринг: 6.8/10

| Критерий | Балл | Обоснование |
|----------|------|-------------|
| **Размер рынка** | 7/10 | 622K фермеров, но только 10% экспортируют (нужна traceability) |
| **Urgency** | 8/10 | Регулирование обязательно (EUDR дед. 30.12.2025) |
| **Конкуренция** | 6/10 | TraceX, Connecting Food, Siemens предлагают solutions |
| **Цена** | 6/10 | Зависит от объёма (€200-500/месяц для large farms) |
| **Цифровизация** | 5/10 | Низкая среди small/medium farms |
| **Регуляторика** | 9/10 | Обязательно, четкие требования |
| **Техсложность** | 8/10 | Высокая (blockchain, GPS, интеграции) |

**No-Go:** ⚠️ Глобальные конкуренты уже в этом. Ниша — только для export-focused фермеров (10-15% рынка)

## РЕКОМЕНДАЦИЯ #1: PAC Subsidy Automation

### Почему именно это?

1. **Огромный рынок:** 622K фермеров, €4.8B бюджета ежегодно
2. **Нет конкурентов:** Gestoras = ручной труд, нет SaaS (!)
3. **Регуляторная поддержка:** EU требует электронизации
4. **Быстрый запуск:** 4-6 недель MVP
5. **Стабильный доход:** Государственная программа, не зависит от цикла
6. **Pain очевидна:** Фермеры теряют деньги на ошибках, платят комиссии

### MVP (недели 1-6)

**Неделя 1-2: Планирование + дизайн**
- Интервью 5-10 фермеров (боль, процесс, готовность платить)
- Интервью 2-3 gestoras (поймёшь, что их раздражает)
- Чтение документации CAP (на испанском)

**Неделя 3-4: Минимум**
- Web-форма на испанском
- Guided wizard (20 вопросов → заявка)
- Email-отправка (PDF или интеграция с FEGA API, если доступен)
- Landing page + Google Ads (€500 бюджета)

**Неделя 5-6: Валидация**
- 20 бета-юзеров (фермеры, найти через Linkedin + WhatsApp)
- Feedback, итерация
- Pre-sales: сколько готовы платить?

### CustDev вопросы

```
1. "Как часто вы подаёте заявку на CAP-субсидии?" → frequency
2. "Сколько вы платите gestoras за подачу?" → LTV
3. "Какие документы вам нужны для заявки?" → scope
4. "Были ли отказы или задержки?" → pain
5. "Вы бы платили €300-500/год за автоматизацию?" → willingness
6. "Вы используете WhatsApp для бизнеса?" → channel preference
```

### Готовность платить

- **Малые фермеры (< 10 га):** €250-400/год (экономят 2-3% от субсидии)
- **Средние фермеры (10-50 га):** €400-600/год
- **Крупные фермы (> 50 га):** €600-1000/год

**LTV (5 лет):** €1250-5000 за фермера

## РЕКОМЕНДАЦИЯ #2: Wine Label Compliance SaaS

### Почему это работает?

1. **Регулирование уже в силе:** QR-коды обязательны с дец. 2023
2. **Болезненный процесс:** Бодеги не знают, как реализовать
3. **Штрафы:** До €50K (мощный мотиватор)
4. **Нет локального конкурента:** Scantrust/QRFox — глобальные, дорогие
5. **Простой MVP:** Дизайнер + веб-разработчик, 3-4 недели

### MVP (недели 1-4)

**Неделя 1: Исследование**
- Найти 5-10 бодег, спросить:
  - "Как вы реализовали QR-коды?"
  - "Сколько вы потратили?"
  - "Готовы ли вы платить за автоматизацию?"

**Неделя 2: Дизайн**
- Figma макет: upload форма → preview этикетки → QR-генератор

**Неделя 3: Код**
- Backend: NodeJS + Redis (для кеша)
- Frontend: React
- QR library: `qrcode.react` или похожий
- PDF-генератор: `pdfkit`

**Неделя 4: Beta**
- 10-15 бодег, feedback, iterate

### Почему Испания так отстаёт в AgriTech?

1. **Mentality:** Фермеры консервативны, используют методы дедов
2. **Возраст:** Средний возраст фермера ~60 лет
3. **Размер ферм:** 51% < 5 га, сложно масштабировать решения
4. **Интернет:** 30% не имеют нормального интернета (сельская местность)
5. **Язык:** Локальный язык обязателен (Испанский + местные диалекты)

### Как это преодолеть?

- **WhatsApp-first:** Не email, не call, а WhatsApp
- **Демонстрация:** Показать результаты на соседней ферме
- **Локальный:** Partnship с agronomos (агрономов), кооперативами
- **Бесплатно:** Пилот бесплатно, потом платный
